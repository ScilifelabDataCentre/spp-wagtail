# Workflow changes need team discussion

**Decision to be made:** What is the procedure when a fix or feature could change how the team works?

**Logic:** If an issue fix changes an established workflow or way of working, implementing it without team awareness can cause longer debugging, conflicting approaches, or surprise in PR review. Workflow-impacting ideas should be visible before merge.

**Status:** decision

**Decision:** Tasks or PR ideas that could change workflow are discussed at the next sprint meeting (daily or other team sync). Create a decision document in this folder, reference it in the PR, and share the link on Slack for anyone not in the meeting.

**Where can I find the outcome of this decision?**

- This `decisions/` folder
- [Developer conventions — process](../dev-conventions.md#process-and-pull-requests)
- PR descriptions that link a decision file
- Team Slack / FREYA process notes
